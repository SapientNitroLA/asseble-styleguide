Making UI documentation a bit sweeter

Sugarcoat was created to enable developers to produce rich UI documentation easily and with minimal up-keep. Sugarcoat works by parsing project files for documentation comments (similar to JavaDoc, JSDoc, etc.) and generates HTML or JSON that is organized and easy to read. Sugarcoat allows developers and designers to access up-to-date previews of UI elements, page components, project specific colors and typography, all in one place.

View this site's Pattern Library

Features

Universal Comment Styles

Use the same commenting syntax (JSDoc) across all file types! If you're writing HTML, just wrap your comments in a standard HTML comment block

Easy-to-identify component states

You can tell sugarcoat that there are modifier states in your css, right within your comment block! Sugarcoat will highlight them for extra readability

Decide where your library lives

Sugarcoat will never impose a folder structure on you

Prefixed Variables? No Problem.

Sugarcoat will still understand your variables if they're SCSS, LESS, or even CSSNext future specs

No Style Bleed

No iFrames means no re-calculating dimensions for component example containers.
